# 启动脚本改进总结 (2026-05-13)

## 📝 改进概览

本次改进了 ORCP 项目的所有启动脚本，提升了用户体验、错误处理和诊断能力。

---

## ✨ 改进清单

### 1. **setup.bat** (Windows 安装脚本)

#### 主要改进
- ✅ **详细进度显示**：8 步安装过程，每步都有清晰编号
- ✅ **日志记录**：所有操作都记录到 `install.log`
- ✅ **错误处理**：每个关键步骤都有失败检测和回退机制
- ✅ **cuDNN 8 支持**：自动检测和安装指导
- ✅ **GPU 自动选择**：检测 NVIDIA 显卡，自动选择 GPU/CPU 模式
- ✅ **FFmpeg 自动下载**：失败时提示手动安装
- ✅ **虚拟环境管理**：支持 `--reinstall` 清理旧环境
- ✅ **帮助信息**：`-h` 或 `--help` 显示完整用法

#### 代码质量
- 改进了 PowerShell 调用的可靠性
- 更好的文件路径处理 (避免路径问题)
- 清晰的子程序结构 (`:help`, `:log` 等)

#### 新增命令行选项
```batch

setup.bat --cpu          # 强制 CPU 模式
setup.bat --gpu          # 强制 GPU 模式
setup.bat --no-ffmpeg    # 跳过 FFmpeg 安装
setup.bat --reinstall    # 清理并重新安装
setup.bat -h / --help    # 显示帮助
```

### 5. **diagnose.bat** (新增 - Windows 诊断脚本)

#### 功能
1. **系统环境检查** - OS, 版本, 用户
2. **Python 环境检查** - 版本, 路径, 可用性
3. **包管理工具检查** - uv, pip 状态
4. **关键依赖检查** - FFmpeg, Git
5. **GPU/CUDA 检查** - NVIDIA GPU, 驱动版本
6. **cuDNN 8 检查** - GPU ASR 加速库
7. **Python 包检查** - 虚拟环境, 依赖状态
8. **文件系统检查** - 项目结构完整性
9. **日志检查** - install.log 最后 5 行
10. **启动建议** - 基于诊断结果的操作建议

#### 用途
- 快速诊断问题根源
- 验证环境配置正确性
- 报告问题时提供系统信息

## 📋 使用建议

### 首次用户
1. 运行 `setup.bat` (Windows) 或 `bash setup.sh` (Linux)
2. 选择 GPU 模式或 CPU 模式
3. 等待安装完成
4. 运行 `orcp_gui.bat` (Windows) 或 `bash orcp_gui.sh` (Linux)

### 遇到问题
1. 运行 `diagnose.bat` (Windows) 或 `bash diagnose.sh` (Linux)
2. 查看诊断报告和建议
3. 按照建议操作
4. 查看 `install.log` 获取更多详情

### 重新安装
1. 运行 `setup.bat --reinstall` (Windows) 或 `bash setup.sh` 后手动删除 `.venv` (Linux)
2. 重新启动应用

## 🔧 技术细节

### 脚本特性
- **Batch (Windows)**：使用 `setlocal enabledelayedexpansion` 处理复杂变量
- **Bash (Linux)**：使用 `set -euo pipefail` 确保脚本安全
- **日志记录**：所有脚本都支持日志输出
- **错误处理**：多层次的错误检测和回退机制

### 兼容性
- **Windows**：Windows 7+ (需要 PowerShell)
- **Linux**：Ubuntu, Fedora, Arch, openSUSE, Debian 等
- **Python**：3.11+ 要求

### 依赖
- Python >= 3.11
- uv 包管理器
- FFmpeg (视频处理)
- NVIDIA GPU (可选，GPU 加速)
